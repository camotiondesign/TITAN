# Carousel Layout Patterns

## Proven Layout Patterns from Top Performers

Based on analysis of top-performing carousels (by impressions and engagement rate), these layout patterns consistently drive performance.

---

## Pattern 1: Side-by-Side Comparison (Highest Impressions)

**Performance:**
- Blog image: 1,539 impressions, 5.5% engagement
- Your Pharmacy Isn't Struggling: 2,525 impressions, 53.4% engagement

**Layout Structure:**
- Vertical split screen (left/right panels)
- Strong visual contrast between sides
- Title/headline at top of each panel
- Main graphic (bars, charts) in center
- Supporting elements (icons, figures) positioned around
- Clean divider between sides

**Visual Elements:**
- Left side: Old model (red/problem colors, chaotic, single element)
- Right side: New model (green/solution colors, organized, multiple elements)
- Color coding: Red = problem, Green = solution
- Figures: Passive (old) vs confident (new)

**When to Use:**
- Model comparisons (dispensing-led vs service-led)
- Before/After transformations
- Problem vs Solution narratives
- Old system vs New system

**Example:**
- "Before" panel: Single tall "Dispensing" bar, passive figure
- "Now" panel: Multiple service bars, confident professional

## Layout Principles

### Visual Hierarchy
1. **Hook Slide**: Strongest visual contrast, boldest design
2. **Problem Slides**: Color-coded (red), visual metaphors
3. **Solution Slides**: Product UI, clean visuals, organized
4. **Proof Slides**: Real photos, testimonials, quotes
5. **CTA Slides**: Clean, focused, brand colors

### Slide Progression
- **Clear Narrative Flow**: Each slide builds on previous
- **Visual Consistency**: Maintain style throughout
- **Progressive Revelation**: Build understanding slide by slide
- **Strong Contrast**: Hook slide must stop scroll

### Background Patterns
- **White/Clean**: Most common, professional
- **Gradients**: Blue (info), white-to-light-blue (calm)
- **Color-Coded**: Red (problem), Green (solution)
- **Product UI**: Titanverse purple console theme

---

## Typography in Carousels

### Headlines
- **Font**: Bold, sans-serif
- **Size**: Large, prominent
- **Color**: Black or white (depending on background)
- **Style**: Uppercase for emphasis, title case for readability

### Body Text
- **Font**: Clean sans-serif
- **Size**: Readable, not too small
- **Color**: Black or dark gray
- **Style**: Clear, scannable

### On-Slide Text
- **Placement**: Prominent, doesn't compete with visuals
- **Hierarchy**: Headline > Subhead > Body
- **Contrast**: Ensure readability against background

---

## Color Usage in Carousels

### Problem Slides
- **Primary**: Red (problems, X marks)
- **Background**: White or light gradient
- **Text**: Black or dark gray

### Solution Slides
- **Primary**: Green (solutions, checkmarks)
- **Accent**: Titanverse purple (product UI)
- **Background**: White or light gradient

### Hook Slides
- **Contrast**: Strong visual contrast (side-by-side)
- **Colors**: Vary by message (red/green for problem/solution, brand colors for product)
- **Background**: White or light gradient

### Product Slides
- **Primary**: Titanverse purple (console UI)
- **Supporting**: Teal-green, white
- **Background**: White or product UI mockup

---

## Best Practices

### Do
- Use strong visual contrast on hook slide
- Maintain consistent style throughout carousel
- Reference top-performing visual patterns
- Use color coding for problem/solution
- Show product UI for feature demonstrations
- Use real photos for case studies
- Keep backgrounds clean and uncluttered

### Don't
- Mix too many visual styles in one carousel
- Use cluttered backgrounds
- Overcomplicate layouts
- Ignore proven patterns from top performers
- Use too many colors (stick to palette)
- Make text too small or hard to read
